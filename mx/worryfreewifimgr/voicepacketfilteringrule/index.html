Used to enter one or more filtering rules (SIP, TCP, etc., using the format "protocol:port") to specify which voice protocols to analyze, excluding all others. If no protocols are specified, all voice packets are analyzed by default.

Parm value input rules:

  • String from 1–64 characters in the format "protocol:port"
  • Separate multiple entries with commas (i.e. “protocol1:port1, protocol2:port2, protocol3:port3”)
